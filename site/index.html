MCP Server · v0.1.0 · Windows · Linux · macOS

Ask your AI assistant what's happening on this machine.

UMI is a Python MCP server that gives AI assistants live access to local system state — disk, memory, processes, services, network, and logs — across Windows, Linux, and macOS. Install it once. Ask questions in plain language.

Memory pressure is the main concern. The service crashes are worth investigating.

02Tools

Seven MCP tools. Each returns structured JSON conforming to the UMI schemas.

get_umi_uptime
Hostname, OS, architecture, boot time, uptime, CPU count, total RAM, current CPU utilization, memory used/available, swap, load averages
get_umi_disk
Mounted volumes — capacity, used/free bytes, filesystem type, I/O counters (reads, writes, bytes, time)
get_umi_network
Network interfaces — IPs, MAC, link speed, status, cumulative bytes/packets sent and received, errors, drops
get_umi_process
Running processes sorted by CPU. Filter by name substring or limit to top N. Returns memory, status, owner, command line, thread count
get_umi_service
System services — Windows Services, systemd units, launchd agents — with status, start type, binary path, user context
get_umi_user
Local user accounts — UID, home directory, shell, group memberships, admin status
get_umi_events
Recent system log entries — Windows Event Log, Linux journald, macOS unified log. Filter by level, source, and count. Messages truncated at 500 chars

03Install

Requires Python 3.10+ and an MCP-compatible AI client.

1. Install the package
pip install git+https://github.com/spinchange/umi.git#subdirectory=mcp-server
2. Add to claude_desktop_config.json Windows: %APPDATA%\Claude\claude_desktop_config.json macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
{ "mcpServers": { "umi": { "command": "python", "args": ["-m", "umi_mcp"] } } }
3. Restart Claude Desktop, then verify
# Settings → Developer → UMI should show as connected # Then ask: "Use get_umi_uptime and tell me the result"

04Supported clients

UMI works with any MCP-compatible client. The config format varies — check your client's MCP documentation.

Client Config location Status
Claude Desktop claude_desktop_config.json Confirmed
Cursor .cursor/mcp.json Confirmed
Windsurf MCP settings in config Confirmed
Cline (VS Code) cline_mcp_settings.json Confirmed
VS Code Copilot Agent Mode MCP settings Confirmed
Codex CLI ~/.codex/config.toml Confirmed
Qwen Code MCP settings in config Confirmed

Note: coding assistants with native shell access (Cursor, Codex, etc.) benefit less from UMI than chat-oriented clients like Claude Desktop, which have no shell access by default.
